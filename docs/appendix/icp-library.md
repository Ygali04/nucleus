# Canonical ICP Library

A reference library of 24 canonical Ideal Customer Profiles (ICPs)
that Nucleus uses as defaults for new tenants. Each ICP is a
starting point; tenants customize from here.

The ICPs are organized by functional role because that's how
brand-marketing teams segment their audiences in practice.

## How to read this page

Each ICP has:

- **Role / title**
- **Company context** (size, type, stage)
- **Pain points**
- **Objections they raise**
- **Preferred tone**
- **Preferred content format**
- **Common languages**

Nucleus's generator agent uses all of these to tailor scripts. An
ICP with "skeptical, analytical" tone produces different output
than an ICP with "enthusiastic, aspirational" tone, even for the
same product and same language.

## Using this library

For each new tenant, the recommended onboarding flow is:

1. **Start with 3–5 ICPs from this library** that most closely
   match the tenant's target customers
2. **Let the tenant customize** — tone, language, specific pain
   points, objections
3. **Run the Brand KB auto-suggest** — often surfaces 2–3 more
   ICPs specific to the tenant's business
4. **Refine based on in-market performance** — after 2 weeks of
   jobs, the Nucleus analytics show which ICPs produce the best
   neural scores and in-market results; re-weight accordingly

## What this library is not

- **Not exhaustive.** A tenant might need an ICP that isn't here.
  That's fine — tenants can create their own from scratch.
- **Not static.** ICPs evolve. This library is updated quarterly
  based on actual tenant usage patterns.
- **Not a recommendation to pick all 24.** Most tenants use 3–8
  active ICPs. More than 10 becomes unmanageable.
- **Not a replacement for customer research.** A tenant should
  still talk to their actual customers. The library is a
  starting point, not a substitute.
